# setup-ssh-hardening

Скрипт автоматической настройки и усиления безопасности Linux-сервера: создание пользователя с sudo, смена порта SSH, UFW, принудительный вход по ключу, отключение паролей и root, fail2ban и опциональная установка Netbird.

## Что делает скрипт

1. **Создание пользователя** — запрос имени и пароля, создание учётной записи и добавление в группу `sudo` (или `wheel` на RHEL).
2. **UFW** — сброс правил; разрешены только выбранный порт SSH (TCP) и 443 (HTTPS).
3. **sshd_config** — смена порта SSH и ужесточение:
   - `PubkeyAuthentication yes` — принудительная авторизация по ключу
   - `PasswordAuthentication no`, `ChallengeResponseAuthentication no`, `UsePAM no`
   - `PermitRootLogin no`, `PermitEmptyPasswords no`
4. **systemd** — настройка `ssh.socket` на новый порт.
5. **fail2ban** — установка и настройка jail для SSH: `bantime = 604800` (неделя), `findtime = 86400` (сутки), `maxretry = 3`, `mode = aggressive`.
6. **Netbird** (опционально) — установка и регистрация по setup-ключу (UUID).
7. **Проверка переопределений** — в конце выводится список файлов в `/etc/ssh`, где заданы ключевые параметры SSH (на случай переопределения через `Include` или другие конфиги).

Поддерживаются дистрибутивы с **systemd**, **UFW** и пакетными менеджерами: **apt** (Debian/Ubuntu), **dnf** / **yum** (RHEL/Fedora и др.).

## Меню выбора

При запуске скрипт показывает меню — можно выполнить одно действие, несколько (номера через пробел) или полный сценарий:

| № | Действие |
|---|----------|
| 1 | Создать пользователя (с паролем и sudo) |
| 2 | Настроить политики SSH (порт, ключ, отключение паролей/root) |
| 3 | Настроить UFW (только SSH + 443) |
| 4 | Установить и настроить fail2ban |
| 5 | Установить Netbird |
| 6 | Проверить переопределение параметров SSH в `/etc/ssh` |
| 7 | **Выполнить всё по порядку** (полный сценарий) |
| 0 | Выход |

Примеры: ввод `1 2 4` — создать пользователя, настроить SSH и установить fail2ban; ввод `7` — полный сценарий (как раньше, один раз все шаги с подтверждением).

## Требования

- Запуск с правами root: `sudo`
- Перед отключением входа по паролю **обязательно** скопировать SSH-ключ новому пользователю (`ssh-copy-id -p PORT user@server`), иначе возможна потеря доступа к серверу

## Быстрый запуск

Запуск одной командой (скрипт скачивается и выполняется):

```bash
sudo bash <(curl -sL https://raw.githubusercontent.com/s-bug0ff/setup-ssh-hardening/main/setup-ssh-hardening.sh)
```

При проблемах с IPv6 можно принудительно использовать IPv4:

```bash
sudo bash <(curl -sL --ipv4 https://raw.githubusercontent.com/s-bug0ff/setup-ssh-hardening/main/setup-ssh-hardening.sh)
```

При выборе **отдельных пунктов** скрипт запросит только нужные данные (например, для пункта 2–4 — порт SSH; для 1 — имя и пароль пользователя).

При выборе **полного сценария (7)** по очереди запрашиваются: имя пользователя, пароль, порт SSH, подтверждение, установка Netbird (и ключ при необходимости).

## Установка вручную

```bash
git clone https://github.com/s-bug0ff/setup-ssh-hardening.git
cd setup-ssh-hardening
sudo ./setup-ssh-hardening.sh
```

## Предупреждение

После применения настроек вход по паролю и под root будут отключены, разрешён только вход по ключу. **До закрытия текущей сессии** скопируйте новому пользователю SSH-ключ и проверьте подключение на новый порт в отдельной сессии:

```bash
ssh-copy-id -p PORT newuser@server
ssh -p PORT newuser@server
```
